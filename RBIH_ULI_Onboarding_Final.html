01
Step One
IP Whitelisting
⏱ 2–3 Business Days

All access to the ULI portal and APIs is restricted to formally whitelisted IP addresses. Lenders must submit a complete security documentation package for review. The TAT clock begins only once all documentation is received in the correct format.

1. Network / Architecture Diagram
  • ✦Must include CBS, LOS, LMS and all systems connecting to ULI — with the integrating system clearly highlighted
  • ✦Data-flow diagram showing communication pathways with the ULI environment
  • ✦Written explanation covering API interactions, data flow, and architecture justification
  • ✦Separate diagrams for Production and DR environments (or clearly labelled on a single diagram)
2. Anti-Malware Solution Evidence
  • ✦Solution name with screenshots — service active, definitions updated, latest scan results
  • ✦Ransomware protection capability must be explicitly stated in the submission
  • ✦Coverage list mapping all servers with anti-malware to the architecture diagram
  • ✦Explicit confirmation of coverage for both Production AND DR environments
  • ✦If cloud-hosted: written justification of how threats are mitigated in the cloud environment
3. VAPT Report
  • ✦Conducted by a CERT-IN empanelled auditor — must be explicitly stated within the report
  • ✦Must include both Vulnerability Assessment (VA) and Penetration Testing (PT)
  • ✦Report less than 6 months old at time of submission; valid for one year for permanent whitelisting
  • ✦All findings closed with a formal closure / clearance report — low-severity findings acceptable only if confirmed low-impact
  • ✦Must confirm the system tested is the system integrating with ULI; IPs must match, or a written explanation provided
  • ✦Separate VAPT reports required for Production and DR environments
02
Step Two
UAT Credentials
⏱ 1 Business Day

Once IP whitelisting is confirmed, RBIH provisions UAT credentials enabling login to the ULI developer portal, API subscription, and integration testing in the UAT environment.

Information Required
  • ✦Full Name — Authorised user / designated SPOC for ULI access
  • ✦Email ID — Official business email address
  • ✦Phone Number — Active mobile number
  • ✦Department — User's department within the organisation
Credential Security Protocols
  • ✦Credentials delivered exclusively via secure one-to-one email — no group emails or chat platforms
  • ✦Utmost discretion required when sharing credentials internally; sanctity must be maintained at all times
  • ✦Any suspected breach must be reported to the RBIH onboarding SPOC immediately
03
Step Three
Integration & Testing
⏱ 1–2 Weeks

With UAT credentials in hand, lenders integrate ULI APIs into their systems. RBIH provides hands-on guidance throughout. A formal written sign-off is mandatory before the Go-Live review can commence.

Integration Process
✦Subscribe to APIs
Log in to the ULI developer portal and subscribe to all APIs required for your lending workflow; the API catalogue and documentation will be provided by the onboarding team
✦Technical Integration
Implement subscribed API calls within your LOS, LMS, or CBS per API specifications; the RBIH onboarding team is available for technical assistance at every step
✦End-to-End Testing
Conduct thorough testing of all integrated flows; validate API responses, error handling, and data flows against expected outcomes; raise all blockers promptly
✦Sync-Up Calls
Schedule calls with the RBIH onboarding team at any point to track progress and resolve technical challenges
Mandatory UAT Sign-Off
✦Confirmation Email
Send a formal email to lender-onboarding@rbihub.in explicitly stating that integration testing is complete and UAT sign-off is confirmed
✦Triggers Go-Live
This email is a mandatory prerequisite — it triggers the Go-Live review process and must be sent before production access can be considered
04
Step Four
Go-Live
Upon MSA Execution

Following receipt of the UAT sign-off, RBIH initiates the go-live review. All prerequisites below must be satisfied before production credentials are issued and the lender is officially live on the ULI platform.

Go-Live Prerequisites Checklist
✦UAT Sign-Off
Written confirmation received at lender-onboarding@rbihub.in stating UAT integration is complete and sign-off is granted
✦MSA Executed
Master Service Agreement between the lender and RBIH fully executed and closed by both parties prior to go-live
✦Commercial Sign-Off
Pricing and commercial terms agreed and signed off with applicable service providers where service provider commercials are relevant to the integration
✦Go-Live Review
RBIH team verifies all three criteria above are satisfactorily met before production access is approved
✦Production Credentials
Once all prerequisites are confirmed, a go-live email with production credentials is issued directly to the designated SPOC
