Step 01 of 03

IP Whitelisting
Prerequisites

Before any access is granted to the ULI platform, submit this security documentation package. All three documents are mandatory.

Three Documents Required

The RBIH security team reviews all three before whitelisting your IPs. Ensure every item below is addressed — missing even one will halt the process.

01
📐
Network & Architecture Diagram
Infrastructure Documentation
  • ✦Must include CBS, LOS, LMS and all systems connecting to ULI — with the integrating system clearly highlighted
  • ✦Data-flow diagram showing communication pathways with the ULI environment
  • ✦Written explanation covering API interactions, data flow and architecture justification
  • ✦Separate diagrams for Production and DR environments, or clearly labelled on one diagram
  • ✦Callback URLs and IP addresses of the source module that will connect to ULI
⚠ All IPs in the diagram must match exactly what connects to ULI — any mismatch requires a written explanation.
02
🛡️
Anti-Malware Solution Evidence
Security Control Documentation
  • ✦Solution name with screenshots — service active, definitions updated, latest scan results shown
  • ✦Ransomware protection capability must be explicitly stated in the submission
  • ✦Coverage list mapping all servers with anti-malware to the architecture diagram
  • ✦Explicit confirmation of coverage for both Production AND DR environments
  • ✦If cloud-hosted: written justification of how external threats are mitigated
⚠ Required regardless of whether infrastructure is on-premise or cloud-hosted.
03
🔍
VAPT Report
Vulnerability & Penetration Testing
  • ✦Conducted by a CERT-IN empanelled auditor — must be explicitly stated within the report
  • ✦Must include both VA (Vulnerability Assessment) and PT (Penetration Testing)
  • ✦Report must be less than 6 months old at time of submission
  • ✦All findings closed with a formal closure/clearance report — low-severity only if confirmed low-impact
  • ✦IPs tested must match the IPs that will connect to ULI, or a written explanation provided
  • ✦Separate reports required for Production and DR environments
⚠ Report is valid for 1 year for whitelisting purposes. All critical findings must be closed — no exceptions.

Step 03 of 03

WSO2 Developer Portal Guide

Log in with your UAT credentials, generate sandbox keys, and subscribe to the ULI APIs you need for integration testing.

🌐

Access the ULI Developer Portal

Use the credentials issued to you after IP whitelisting was confirmed ↑ See Step 01. The portal gives you access to sandbox credentials, token generation, and API subscriptions.

am.nonprod.rbihub.io/devportal ↗ 📖 API Documentation ↗
01
🔑
Login
Sign In with Your UAT Credentials
Navigate to am.nonprod.rbihub.io/devportal and log in using the credentials shared with you after IP Whitelisting was confirmed ↑ Step 01.

These credentials were delivered to your registered email address via a secure one-to-one email from the RBIH onboarding team. If you have not received them, verify that your IP whitelisting is complete and contact your onboarding SPOC.
02
📋
Navigation
Open the Applications Tab
Once logged in, you will see the list of available ULI APIs on the screen. Look at the top navigation bar — click on the Applications tab to proceed to your lender workspace.
03
🏢
Your Application
Select Your Lender Application
Inside the Applications tab, you will see your lender name / department listed as an application. Click on it to open your organisation's dedicated application workspace where you manage credentials and API subscriptions.
04
🗝️
Sandbox Credentials
Access Your Sandbox Credentials
You will be redirected to a new window. On the left side menu, click on Sandbox Keys. This gives you access to your sandbox credentials for the non-production environment:

  1. Client ID — your application identifier for the non-production environment
  2. Client Secret — your application secret key
  3. To generate an access token for API testing, click the Generate Token button on this screen
05
🔗
API Subscriptions
Subscribe to ULI APIs
From the same left side menu (as seen in Step 04 above), select the Subscriptions tab. You will see a summary of all APIs you have subscribed to so far and their current status.

To subscribe to a new API:
  1. Click the + Subscribe APIs button at the top of the screen
  2. Search for the API by name — refer to docs.rbihub.in for the full list
  3. Click the Subscribe button next to the API you need

After subscribing — contact your onboarding SPOC to request approval. Subscriptions only become active once approved by the RBIH team.
⚠ Post-Subscription: Approval Required
After subscribing to APIs, contact your RBIH onboarding SPOC to request approval before proceeding with integration testing. Subscriptions must be activated by the RBIH team. Reach out at lender-onboarding@rbihub.in — sync-up calls can be arranged at any stage.
